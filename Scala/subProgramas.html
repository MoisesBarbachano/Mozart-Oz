Subprogramas

Bloques

Puedes combinar expresiones con “{ }”. Y a esto le podemos llamar bloque. El resultado de la última expresión en el bloque es el resultado de todo el bloque en general.

println({

val x = 1+1

x + 1

})

Funciones

Las funciones son expresiones que pueden tomar parámetros. Puedes definir una función anónima(sin nombre) que regrese un entero más uno:

(x: Int) => x + 1

Por la izquierda de => hay una lista de parámetros. A la derecha es una expresión que envuelve los parámetros.

Igualmente puede nombrar a las funciones:

val addOne = (x: Int) => x + 1

println(addOne(1)) // 2

Las funciones pueden tomar múltiples parámetros:

val add = (x: Int, y : Int) => x + 1

println(add(1,2)) // 3

O pueden no tomar parámetros:

val getTheAnswer = () => 42

println(getAnswer()) // 42

Métodos

Los métodos lucen y se comportan de manera similar a las funciones, pero hay unas pocas diferencias clave entre ellos. Los métodos son definidos con la palabra clave “def”. “def” es seguido por un nombre, lista de parámetros, tipo de retorno y un cuerpo.

def add(x: Int, y: Int ): Int = x + y

println(add(1,2))

Observa como el tipo de retorno es declarado después de la lista de parámetros y “: Int”.

Los métodos pueden tomar múltiples listas de parámetros:

def addThenMultiply(x: Int, y: Int)(multiplier: Int): Int = (x + y) * multiplier

println(addThenMultiply(1, 2)(3)) // 9

O ninguna lista de parámetros:

def name:String = System.getProperty("user.name")

println("Hello, " + name + "!")

Los métodos pueden ser expresiones multilineas

def getSquareString(input: Double): String = {

val square = input * input

square.toString

}

La última expresion en el cuerpo es el tipo de retorno de valor del método. (Scala tiene un “return”, pero rara vez es usado).

Clases

Puedes definir clases con la palabra clave seguida por su nombre y parámetros de constructor.

class Greeter(prefix: String, suffix: String) {

def greet(name: String): Unit =

println(prefix + name + sufix)

}

Tú puedes hacer una instancia de la clase con la palabra clave “new”.

val greeter = new Greeter("Hello, ", "!")

greeter.greet("Scala developer") // Hello, Scala developer!

Case clases

Scala tiene un tipo especial de clase llamado un “case” class. Por defecto, las case clases son inmutables y comparadas por valor. Tú puedes definir case clases con la palabra “case class”.

case class Point(x: Int,y: Int)

Puedes instanciar las case clases sin la palabra clave “new”.

val point = Point(1, 2)

val anotherPoint = Point(1, 2)

val yetAnotherPoint = Point(2, 2)

Objetos

Los objetos son instancias simples de sus propias definiciones. Puedes pensar en ellos como singletons de sus propias clases.

Puedes definir objetos con la palabra clave objeto.

object IdFactory{

private var counter = 0

def create(): Int = {

counter += 1

counter

}

}

Puedes tener acceso a un objeto refiriéndose por su nombre.

val newId: Int = IdFactory.create()

println(newId) // 1

val newerId: Int = IdFactory.create()

println(newerId) // 2

Hilos

Traits son tipos que contienen ciertos campos y métodos. Múltiples traits pueden ser combinados.

Puedes definir traits con la palabra clave “trait”.

trait Greeter{

def greet(name:String): Unit

}

Los Traits también pueden tener implementaciones por default.

trait Greeter{

def greet(name:String): Unit=

println("Hello," + name + "!")

}

Puedes extender traits con la palabra clave extends y anular una implementación con la palabra clave “override”.

Método Main

El método main es un punto de entrada del programa. La máquina virtual de Java requiere un método main para ser llamado “main” y toma un argumento, un array de strings.

Usando un objeto, puedes definir un método main como el siguiente:

object Main{

def main(args: Array[String]): Unit =

println("Hello, Scala developer!")

}
