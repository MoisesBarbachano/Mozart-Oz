Expresiones y Asignaciones

En Scala los operadores son métodos. Cualquier método con un simple parametro puede ser usado como un operador infijo.

Por ejemplo, + puede ser llamado con punto-notación: 10.+(1)

De cualquier forma, es más fácil leerlo como un operador infijo: 10 + 1

Definiendo y usando operadores

Puedes usar cualquier identificador legal como un operador. Esto incluye un nombre como “add” o símbolos como “+”.

Ejemplo:

case class Vec(val x: Double, val y: Double) {

def +(that: Vec) = new Vec(this.x + that.x, this.y + that.y)

}

val vector1 = Vec(1.0, 1.0)

val vector2 = Vec(2.0, 2.0)

val vector3 = vector1 + vector2

vector3.x // 3.0

vector3.y // 3.0

La clase Vec tiene un método “+” el cual usamos para sumar vector1 y vector2.

Usando paréntesis, puedes construir expresiones complejas con sintaxis que pueda ser leída.

Aquí está la definición de la clase MyBool la cual incluye métodos “and” y “or”.

case class MyBool(x: Boolean) {

def and(that: MyBool): MyBool = if (x) that else this

def or(that: MyBool): MyBool = if (x) this else that

def negate: MyBool = MyBool(!x)

}

Ahora es posible usar “and” y “or” como operadores infijos:

def not(x: MyBool) = x.negate

def xor(x: MyBool, y: MyBool) = (x or y) and not(x and y)

Precedencia

Cuando una expresión usa múltiples operadores, éstos son evaluados basados en la prioridad del primer carácter.

* / %

+ -

:

= !

< >

&

^

|

(all letters)

En el siguiente ejemplo podemos ver que ?^ tiene la más alta precedencia porque empieza con el caracter ? . + tiene la segunda mayor precedencia, seguido de ^?, ==> , | y less.

a + b ^? c ?^ d less a ==> b | c

Es equivalente a

((a + b) ^? (c ?^ d)) less ((a ==> b) | c)
