Tipos de Datos

En Scala, todos los valores tienen un tipo, incluyendo funciones y valores numéricos. El diagrama ilustra un subconjunto de la jerarquía de tipos.

Jerarquía de tipos Scala

Any es un supertipo de todos los tipos, también llamado el tipo más alto. Define ciertos métodos universales como equals, hashCode y toString. Any tiene dos subclases directas: AnyVal y AnyRef.

AnyVal representa tipos de valores. Hay 9 tipos de valores predefinidos y son no-nullable: Double, Float, Long, Int, Short, Byte, Char, Unit y Boolean. Unit es un tipo el cual no carga con información significativa. Hay exactamente una instancia de Unit la cual puede ser declarada literalmente como: (). Todas las funciones retornan algo y en ocasiones Unit es usado para retornar tipos.

AnyRef representa tipos de referencia. Todos los tipo de no-valores son definidos como tipos de referencia. Todo tipo de usuario definido en Scala es un subtipo de AnyRef. Si Scala es usado en el contexto del entorno de tiempo de ejecución de Java, AnyRef corresponde al java.lang.Object.

Nothing y null

Nothing es un subtipo de todos los tipos, también es llamado el tipo más bajo. No hay valor que tenga tipo Nothing. Un uso común es para señalar la no-terminación como un lanzamiento de una excepción, salida de programa o un loop infinito.

Null es un subtipo de todos los tipos de referencia. Tiene un valor simple identificado por la palabra clave null. Null es proporcionado en la mayoría de ocasiones para interoperabilidad con otros JVM lenguajes y casi nunca debería ser usado en el código de Scala.

El compilador de Scala a menudo puede inferir el tipo de una expresión sin que se lo declares explícitamente.

Por ejemplo:

val businessName = "Montreux Jazz Café"

El compilador puede detectar que businessName es un String. Funciona de manera similar con metodos:

def squareOf(x:Int)= x*x

El compilador puede inferir que el tipo de retorno es un Int, así que no se requiere especificar el tipo de retorno.
